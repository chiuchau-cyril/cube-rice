## 一、專案背景

Cube Rice 為全自動化外送炒飯廚房，核心特色為「Cube Food」標準化食材系統——所有食材切成 2cm 方塊，搭配 250g 冷凍飯塊，實現全自動烹調出餐。

**營運目標**：
- 24 小時連續運轉
- 日產量 150-220 份（含尖峰可達 30 份/小時）
- 單份出餐時間 ≤ 4 分鐘（含投料、烹調、出鍋）
- 每 15 份或每 30 分鐘自動清洗一次

## 三、自動炒鍋模組規格需求

### 3.1 基本規格

| 項目 | 需求規格 | 備註 |
|------|---------|------|
| 鍋體容量 | 單份 500g 成品（生料含飯約 400g） | 單份出餐 |
| 加熱方式 | 電磁感應加熱（優先）或電熱管 | 無明火需求 |
| 溫控範圍 | 100°C ~ 280°C | 炒飯需高溫鍋氣 |
| 溫控精度 | ±5°C | |
| 翻炒機構 | 行星式攪拌 或 傾斜旋轉 | 需確保飯粒均勻受熱不碎裂 |
| 翻炒轉速 | 可調，建議 30-120 RPM | 不同階段需不同速度 |
| 單次烹調時間 | 180-240 秒 | 含預熱 |
| 鍋體材質 | SUS 304 不鏽鋼 + 不沾塗層內膽 | 食品級 |
| 電源 | 三相 220V 或單相 220V | 視功率需求 |
| 功率 | ≤ 5kW/台（建議） | 需確認 |

### 3.2 烹調程序需求

Cube Rice 的炒飯烹調流程如下，需設備支援全程自動控制：

```
Step 1: 預熱鍋體 → 280°C（30秒）
Step 2: 投入調味油 15ml（自動液體投料）
Step 3: 投入蛋液 40ml（自動液體投料）
Step 4: 快速翻炒（高速 100RPM，15秒）— 炒蛋花
Step 5: 投入冷凍飯塊 250g（自動固體投料）
Step 6: 降溫至 220°C，中速翻炒（60RPM，60秒）— 飯散開
Step 7: 投入方塊食材（雞肉×8顆、蔬菜×6顆等）
Step 8: 升溫至 250°C，中速翻炒（60RPM，60秒）
Step 9: 投入調味料（醬油 5ml、鹽 2g）
Step 10: 高速翻炒（100RPM，30秒）— 收尾
Step 11: 自動出鍋（傾倒至便當盒）
```

**關鍵需求**：
- ✅ 每個步驟的溫度、時間、轉速可獨立設定
- ✅ 支援多種菜色配方存儲（至少 20 種）
- ✅ 配方可透過外部介面更新（不需到場改程式）

### 3.3 連續運轉需求

| 項目 | 需求 |
|------|------|
| 日運轉時間 | 24 小時（含清洗時間） |
| 日產量/台 | 50-75 份 |
| 連續烹調間隔 | ≤ 30 秒（上一鍋出餐到下一鍋開始投料） |
| 設計壽命 | ≥ 5 年（正常使用保養下） |
| MTBF | ≥ 2,000 小時 |

## 四、投料系統規格需求

### 4.1 固體投料（方塊食材）

| 項目 | 需求 |
|------|------|
| 食材形狀 | 2cm × 2cm × 2cm 冷凍方塊 |
| 食材種類 | 雞肉、火腿、紅蘿蔔、豆干等（5-8 種） |
| 計量方式 | 光電計數（優先）或重量計量 |
| 精度 | ±1 顆 |
| 料斗容量 | 各食材 5kg 以上（約 50 份以上） |
| 溫度需求 | 料斗需具備 -5°C ~ 5°C 冷藏功能（防止結塊融化） |
| 防結塊 | 需有振動或攪拌機構防止冷凍方塊黏連 |

**⚠️ 已知挑戰**：冷凍肉塊容易因表面水氣結冰而黏連，需要振動給料 + 分離機構

### 4.2 冷凍飯塊投料

| 項目 | 需求 |
|------|------|
| 飯塊規格 | 250g，10cm × 8cm × 3cm，真空包裝 |
| 投入方式 | 自動拆封（或預拆封後手動放入料斗） |
| 溫度狀態 | 半解凍（表面 0°C，中心 -5°C） |
| 投入精度 | 每次 1 塊 |
| 料斗容量 | 20 塊以上 |

### 4.3 液體投料

| 液體種類 | 單次用量 | 精度 | 備註 |
|----------|---------|------|------|
| 食用油/蔥油 | 10-20 ml | ±2ml | 常溫 |
| 殺菌蛋液 | 30-50 ml | ±3ml | 冷藏 5°C |
| 醬油 | 3-8 ml | ±1ml | 常溫 |
| 水 | 10-30 ml | ±3ml | 常溫 |
| 米酒 | 5-10 ml | ±1ml | 常溫 |

**計量方式**：蠕動幫浦 或 電磁閥 + 流量計

## 五、CIP 自動清洗系統

| 項目 | 需求 |
|------|------|
| 清洗觸發 | 每 15 份 或 每 30 分鐘（先到先清洗） |
| 清洗流程 | 高壓熱水沖洗 → 清潔劑 → 清水沖洗 → 熱風乾燥 |
| 總清洗時間 | ≤ 90 秒 |
| 用水溫度 | 80°C（沖洗）/ 60°C（乾燥熱風） |
| 清潔劑 | 食品級鹼性清潔劑，自動投放 |
| 排水 | 需整合排水管路 |

## 六、物聯網通訊介面需求

### 6.1 必要介面

| 項目 | 需求 | 說明 |
|------|------|------|
| 通訊協定 | Modbus RTU（RS485）或 Modbus TCP | Cube Rice 的邊緣閘道器對接 |

| 可寫入參數 | 啟動/停止、配方選擇、烹調參數覆寫 | 外部控制 |
| 配方管理 | 支援外部上傳/更新配方 | 遠端更新菜色 |

### 6.2 希望有的介面（Nice-to-have）

| 項目 | 說明 |
|------|------|

## 十一、希望洪達回覆的問題

1. **現有機型**：目前 AI 智能炒菜機器人的標準機型規格（容量、功率、加熱方式、尺寸、價格帶）？

5. **CIP 清洗**：現有機型的清洗方式為何？是否可整合自動 CIP？

7. **參考案例**：是否有已商用的餐飲客戶案例可參考？
8. **合作意願**：對上述三種合作模式（採購/共同開發/策略合作），洪達較傾向哪一種？
