## 一、前言

Cube Rice 是一個全自動化外送炒飯廚房專案，核心概念是將所有食材標準化為 2cm 方塊（Cube Food），搭配冷凍飯塊，實現 24 小時無人值守自動出餐。

我們看到洪達自動化近 40 年在自動控制及多軸式機器人的深厚技術底蘊，以及 2023 年推出的「AI 智能全自動炒菜機器人」——電腦控制編程、100+ 菜色配方、全程溫度自控、自動調味投料——與我們的需求高度吻合。

**我們需要的不只是買一台炒菜機，而是找到一個能一起把「自動化炒飯」做到極致的技術夥伴。**

## 三、合作模式方案

### 方案 A：設備採購 + 技術支援（最單純）

```
Cube Rice 向洪達購買炒菜機器人，洪達提供技術支援與客製化服務
```

| 項目 | 內容 |
|------|------|
| 合作方式 | 買賣關係 + 技術服務合約 |
| 設備數量 | 首批 3 台（含 1 台備援容量） |
| 客製化範圍 | 依 Cube Rice 規格書調整投料、通訊介面 |
| 付款方式 | 訂金 30% → 驗收 60% → 保固尾款 10% |
| 保固 | 1 年全機保固，含到場維修 |
| 後續維護 | 年度維護合約（另計） |
| 交期 | 洽談（預估 8-12 週） |

**優點**：簡單明確，風險各自承擔  
**缺點**：客製化深度可能有限，洪達動力較低

---

### 方案 B：共同開發 + 技術授權（推薦）

```
雙方共同開發「Cube Rice 專用自動炒飯工作站」
洪達負責硬體，Cube Rice 負責軟體與配方
```

| 項目 | 內容 |
|------|------|
| 合作方式 | 共同研發合約，明確分工 |
| 洪達負責 | 炒鍋硬體、投料機構、CIP 系統、機電整合 |
| Cube Rice 負責 | 炒飯配方開發、物聯網軟體、營運數據回饋 |
| 開發費用 | 雙方各自承擔各自範圍的開發成本 |
| 設備採購 | Cube Rice 以優惠價採購開發完成之設備 |
| IP 歸屬 | 硬體 IP 歸洪達，軟體 IP 歸 Cube Rice，整合方案雙方共有 |
| 後續擴點 | Cube Rice 展店時優先採購洪達設備 |
| 洪達權益 | 可將此合作案作為商用案例推廣，開拓餐飲自動化市場 |

**優點**：雙方深度合作，產品更貼合需求；洪達獲得餐飲市場示範案例  
**缺點**：開發時程較長，需要更多溝通協調

---

### 方案 C：策略合作 + 營收分潤（深度綁定）

```
洪達以設備入股或租賃方式參與 Cube Rice 營運
共享營運成果
```

| 項目 | 內容 |
|------|------|
| 合作方式 | 策略合作合約 |
| 設備提供 | 洪達以設備出資（折算股份或月租） |
| 營收分潤 | 每月營收 N%（待議）作為設備使用費 |
| 洪達權益 | 直接參與餐飲自動化市場驗證，取得第一手營運數據 |
| Cube Rice 權益 | 大幅降低初期資金壓力 |
| 退出機制 | 合作滿 2 年後，Cube Rice 可選擇買斷設備 |
| 展店合作 | 每新增據點，洪達享有設備優先供應權 |

**優點**：Cube Rice 降低初期投資，洪達深入餐飲場景  
**缺點**：利益分配複雜，需要高度信任

## 四、對洪達的價值主張

### 為什麼洪達應該跟我們合作？

| 價值 | 說明 |
|------|------|
| 🏪 **真實商用場景** | 不是展覽 demo，是真正 24hr 運轉的商業廚房 |
| 📊 **營運數據回饋** | 每日 150+ 份的真實烹調數據，幫助優化設備 |
| 📣 **市場推廣案例** | 「洪達炒菜機器人已實際應用於商業廚房」的最佳行銷素材 |
| 🔄 **持續訂單** | Cube Rice 每展一店就需要 3 台設備，規模化後訂單穩定成長 |
| 🤖 **技術驗證** | 在高強度、高頻率環境下驗證設備穩定性與耐久性 |
| 🍳 **餐飲市場入口** | 透過 Cube Rice 案例，打入團膳、連鎖餐飲等 B2B 市場 |

### 市場潛力

```
單店 3 台 × 目標 5 年 50 店 = 150 台設備需求
加上維護合約、耗材、升級 = 持續性營收
```
