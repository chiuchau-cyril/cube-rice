## 1. 執行摘要

### 專案概述

本計畫提出一個**全自動化外送炒飯廚房**的創新商業模式，透過獨創的「Cube Food」標準化食材系統，結合物聯網智慧製造技術，達成：

- **24 小時無人值守營運**
- **每份餐點 3-4 分鐘自動出餐**
- **食材與品質的極致標準化**
- **可快速複製的單店模型**

### 核心創新

**Cube Food 系統**：將所有食材標準化為 2cm × 2cm × 2cm 的立方體，搭配預分裝冷凍飯塊，讓自動化烹調從「困難」變為「簡單」。

### 關鍵數字

| 項目 | 數值 |
|------|------|
| 總投資額 | **$202 萬** |
| 月固定成本 | **$11.5 萬** |
| 損益兩平 | **105 單/天** |
| 目標日單量 | **150 單/天** |
| 預估年淨利 | **$72 萬** |
| 投資回收期 | **2.8 年** |

### 為什麼是現在？

1. **外送市場持續成長**：台灣外送市場規模突破 800 億，年增率 15% 以上
2. **人力成本攀升**：基本工資連年調漲，餐飲業缺工嚴重
3. **技術已成熟**：自動烹調設備（如美善品）已證明消費級可靠性
4. **消費者接受度提升**：疫情後，消費者對「無接觸」服務接受度大增

## 5. Cube Food 標準化食材系統

### 5.1 概念起源

傳統自動化餐飲的最大障礙在於**食材的不規則性**——每塊肉的大小不同、每根菜的形狀各異，導致自動化設備難以處理。

**Cube Food 系統**從根本解決這個問題：

> 不是讓機器適應食材，而是**讓食材適應機器**。

### 5.2 規格定義

#### 方塊食材標準

| 參數 | 規格 |
|------|------|
| 尺寸 | 2cm × 2cm × 2cm |
| 公差 | ±1%（±0.2mm）|
| 體積 | 8 cm³ |
| 重量 | 依食材密度（約 5-10g）|

#### 冷凍飯塊標準

| 參數 | 規格 |
|------|------|
| 重量 | 250g ±5g |
| 尺寸 | 10cm × 8cm × 3cm（扁平狀）|
| 米種 | 台梗九號 / 壽司米 |
| 含水率 | 58-62% |
| 保存 | -18°C，6 個月 |
| 使用方式 | 半解凍後直接入鍋 |

### 5.3 技術優勢

| 優勢 | 說明 |
|------|------|
| **計數取代秤重** | 8 顆雞肉塊 = 精確份量，比秤重更快更準 |
| **均勻受熱** | 相同尺寸 = 相同烹調時間，品質一致 |
| **庫存簡化** | 以「顆」為單位管理，精確預測 |
| **自動化友善** | 規則形狀方便機械手臂/給料器處理 |
| **供應商標準化** | 明確規格，方便開發供應商和品管 |

### 5.4 冷凍飯塊的關鍵洞察

傳統認知：「熱飯才好吃」

**炒飯真相**：冷飯（隔夜飯）炒出來更粒粒分明、更香

我們的做法：
1. 供應商將米飯煮熟後快速冷卻至 10°C 以下
2. 分裝成 250g 飯塊，急速冷凍
3. 店端半解凍（中心微冰）後直接入鍋
4. 冷飯遇高溫熱鍋 = 完美鍋氣

**結果**：口感更好 + 完全標準化 + 方便自動化

**情境**：炒鍋模組故障，無法出餐

- 設備商快速回應合約（4 小時內）

**緊急處理**：
1. 自動切換至其他炒鍋
2. 通知設備商
3. 若全部故障：暫停接單，通知已下單客戶

#### 風險三：食安事件

**情境**：發生食物中毒或異物投訴

**預防措施**：
- 全程溫度監控（物聯網記錄）
- 食材溯源系統
- 定期第三方檢驗
- 員工健康管理

**緊急處理**：
1. 立即停止營運
2. 保留樣本送檢
3. 配合衛生單位調查
4. 主動聯繫受影響客戶
5. 公開透明處理

- 📊 **全自動數據收集**：用水、用電、來客、訂單，所有營運細節即時掌握
- 🔮 **Digital Twin 模擬**：在虛擬環境中測試決策，降低試錯成本
- 🤖 **AI 驅動營運**：從程式開發到行銷文案，全面擁抱 AI 生成
- 🎛️ **動態參數調整**：根據數據即時調整烹調、定價、行銷策略
