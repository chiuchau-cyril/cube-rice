| 粉體 | 鹽、胡椒 | 粉/顆粒 | 常溫 | 1-5 g | ⭐ |

### 核心技術難點

```
1. 冷凍方塊結塊黏連 → 如何自動分離成單顆？
2. 精準計數/計量 → 不是秤重，是「數」幾顆
3. 時序控制 → 不同食材在不同時間點投入
4. 高溫環境 → 鍋溫 280°C，蒸氣回流會影響投料口
5. 衛生清洗 → 投料管路的 CIP
6. 物理對接 → 投料口與炒鍋的機械介面
```

## 二、固體食材自動投料技術

### 2.1 振動給料器（Vibratory Feeder）

**原理**：利用電磁或機械振動，使食材在料槽中定向移動，控制振動頻率與幅度來調節給料速度。

**食品業應用現況**：
- 廣泛用於 IQF（Individual Quick Frozen）冷凍食品的包裝線
- 知名廠商：Eriez、Syntron、KMG Systems、PROEX FOOD、Alvibra
- 已被證實可處理冷凍蔬菜丁、冷凍肉丁等不規則食材

**關鍵規格參數**：

| 參數 | 典型範圍 | Cube Rice 需求 |
|------|---------|---------------|
| 給料速度 | 10-500 kg/hr | ~50 kg/hr（足夠） |
| 振動頻率 | 30-120 Hz | 依食材調整 |
| 料槽材質 | SUS 304/316 不鏽鋼 | 食品級必須 |
| 溫度環境 | -30°C ~ +60°C | 需耐低溫 |

**優點**：
- ✅ 技術成熟，食品工業標準設備
- ✅ 無活動零件接觸食材（衛生）
- ✅ 可處理冷凍食材
- ✅ 速度可調

**限制**：
- ❌ 難以精確計「顆數」（適合計重）
- ❌ 冷凍方塊如果結塊，振動可能不足以分離
- ❌ 需要搭配其他計量方式

### 2.2 冷凍食品解塊技術（Block Breaker / Cluster Buster）

**這是解決冷凍方塊結塊問題的關鍵技術。**

**PROEX FOOD Cluster Buster System**：
- 專為 IQF 冷凍食品設計的解塊系統
- 使用振動 + 衝擊的組合方式，將黏連的冷凍塊分離成單顆
- 可處理冷凍水果、蔬菜、蛋白質（肉類）
- 兩段式設計：Block Breaker（大塊打散）→ Cluster Buster（小塊分離）

**MTC Cluster Breaker**：
- 用於 IQF 和再冷凍產品
- 可處理整箱（combo）或回溫塊狀產品

**KMG Systems 冷凍食品輸送方案**：
- 專用的表面處理和設計，最小化產品黏附和結塊
- 保持 IQF 個別分離的狀態
- 關鍵技術：**防黏表面塗層** + **受控振動** + **冷氣流維持低溫**

> 💡 **對 Cube Rice 的啟示**：不需要自己發明解塊技術，食品工業已有成熟方案。關鍵是在料斗設計中整合防結塊機構。

### 2.3 計量方式比較

| 計量方式 | 原理 | 精度 | 適用食材 | 成本 | 適合 Cube Rice？ |
|----------|------|------|---------|------|-----------------|
| **重量計量**（Multihead Weigher）| 多頭組合秤，組合出目標重量 | ±0.5-1g | 所有固體 | 高（$50-200K USD）| ⭐⭐⭐⭐ |
| **體積計量**（Volumetric Cup）| 固定容積的杯子盛裝 | ±5-10% | 均勻顆粒 | 低 | ⭐⭐ |
| **光電計數**（Photoelectric Counter）| 光學感測器逐顆計數 | ±1 顆 | 標準化方塊 | 中 | ⭐⭐⭐⭐⭐ |
| **視覺計數**（Machine Vision）| 攝影機 + AI 辨識計數 | ±1 顆 | 任何形狀 | 中高 | ⭐⭐⭐⭐ |
| **螺旋給料**（Screw Conveyor）| 螺旋推送，計量體積 | ±5% | 粉體/小顆粒 | 低 | ⭐（僅限調味料）|

#### Multihead Weigher（組合秤）— 食品工業的黃金標準

**這是最被低估但最適合的技術。**

- 全球知名廠商：Ishida（日本）、Cabinplant（丹麥）、Yamato（日本）、Paxiom（加拿大）
- 原理：將食材分散到 10-24 個小秤盤中，電腦計算哪幾個秤盤的重量組合最接近目標值
- **精度**：±0.5g（遠超過我們 ±1 顆的需求）
- **速度**：每分鐘 30-120 次分配
- 可處理冷凍食品、肉丁、蔬菜丁

> 💡 **思考**：對於 Cube Rice，因為方塊是標準 2cm³，每顆重量約 5-10g，其實可以用**重量計量**間接實現計數（8 顆雞肉 ≈ 64g ±8g）。但真正的光電計數會更直覺。

#### 光電計數 — 最適合標準化方塊的方案

**原理**：食材經振動給料器排成單列，通過光電感測器時逐顆計數。

**實現方式**：
1. 振動給料器 → 將方塊排成單列（關鍵步驟）
2. 窄槽道引導 → 確保一次只通過一顆
3. 對射式光電感測器 → 遮斷光束 = 計一顆
4. 電磁閥擋板 → 達到目標數後關閉

**挑戰**：
- 2cm 方塊可能不容易排成單列
- 冷凍方塊表面有霜，可能影響光學感測
- 需要防止兩顆同時通過（double count）

**解決方案**：
- 使用 V 型振動槽道，自然排列成單列
- 搭配近紅外感測器（不受霜影響）
- 加裝確認感測器（雙重計數校驗）

### 2.4 推薦方案：振動給料 + 光電計數 的整合設計

```
┌─────────────┐     ┌─────────────┐     ┌──────────┐     ┌──────┐
│ 冷藏料斗     │ ──▶ │ 解塊振動器   │ ──▶ │ V型排列   │ ──▶ │ 光電  │
│ (-5°C)      │     │ (分離結塊)   │     │ 槽道     │     │ 計數  │
│ 5kg 容量    │     │              │     │ (單列化) │     │ 閘門  │
└─────────────┘     └─────────────┘     └──────────┘     └──┬───┘
                                                            │
                                                            ▼
                                                      ┌──────────┐
                                                      │ 投入炒鍋  │
                                                      │ (重力/    │
                                                      │  氣動推送) │
                                                      └──────────┘
```

**每種食材一套獨立的給料-計數-投放系統**，由中央控制器統一協調時序。

## 三、液體自動投料技術

### 3.1 蠕動幫浦（Peristaltic Pump）

**原理**：滾輪壓迫柔性管路，液體僅接觸管壁，不接觸泵體，天然適合食品級應用。

**關鍵規格**：

| 參數 | 典型值 | Cube Rice 需求 |
|------|--------|---------------|
| 流量範圍 | 0.0001 ~ 720 ml/min | 10-100 ml/min |
| 計量精度 | **±0.2%**（ATO.com 規格）| ±2ml 即可 |
| 管材 | 矽膠管、Norprene、Tygon | FDA 食品級矽膠 |
| 黏度適應 | 1 ~ 100,000 cP | 蛋液 ~10 cP，醬油 ~2 cP |
| 自吸能力 | 有 | 重要，管路可能較長 |

**優點**：
- ✅ 液體僅接觸管壁，清洗簡單（換管即可）
- ✅ 精度極高，遠超我們的需求
- ✅ 可處理含固體顆粒的液體（如蛋液）
- ✅ 自吸能力強，不需額外引流
- ✅ 成本低（單台 $200-2,000 USD）

**食品級供應商**：
- **PCM（法國）**：Delasco™ 系列，專為食品工業設計
- **Albin Pump**：可處理含固體的食品液體
- **Granzow**：FDA 合規食品級蠕動幫浦
- **Watson-Marlow**：高端食品級蠕動幫浦

**限制與注意**：
- ⚠️ 管路壽命有限（矽膠管 ~500-1000 小時需更換）
- ⚠️ 蛋液需冷藏保存，管路需定期沖洗
- ⚠️ 高溫蒸氣可能從投料口回流，影響管路

### 3.2 電磁閥 + 流量計方案

**原理**：液體儲存在加壓容器中，電磁閥控制開關，流量計量測實際流量。

| 項目 | 說明 |
|------|------|
| 電磁閥 | 食品級不鏽鋼，響應時間 <50ms |
| 流量計 | 科氏力質量流量計 或 橢圓齒輪流量計 |
| 精度 | ±0.5-1% |
| 速度 | 極快（瞬間開關） |

**優點**：反應速度快，適合需要「瞬間」投入的場景（如起鍋前淋醬油）  
**缺點**：管路清洗較複雜，不適合黏稠液體

### 3.3 BotInKit OMNI 的液體投料方案（參考）

BotInKit OMNI 的液體投料系統設計：
- **液體調味盒**（700ml 小 / 1500ml 大）：油、醬油等
- **醬料調味盒**（650ml）：豆瓣醬等黏稠醬料
- **管路泵送**：從調味盒通過管路投入鍋中
- **自動清洗**：鍋和管路都有自動清洗
- ⚠️ **限制**：液體盒只能放純液體，不能有顆粒（會堵塞管路）

**最多可同時裝 13 種調味料**。

### 3.4 Cube Rice 液體投料推薦方案

| 液體 | 推薦方案 | 原因 |
|------|---------|------|
| 食用油/蔥油 | 蠕動幫浦 | 黏度中等，精度需求中 |
| 蛋液 | 蠕動幫浦（大管徑） | 黏稠、含微粒，蠕動幫浦最適合 |
| 醬油 | 電磁閥 + 計時（或蠕動幫浦） | 用量少，需精準 |
| 水 | 電磁閥 + 計時 | 最簡單 |
| 米酒 | 電磁閥 + 計時 | 類似水 |

## 四、冷凍飯塊投料

### 4.1 挑戰分析

冷凍飯塊（250g，10×8×3cm）的投料跟方塊食材完全不同：
- 體積較大，無法用振動給料
- 每次只投一塊，不需計數
- 需要在「半解凍」狀態投入（表面 0°C，中心 -5°C）
- 可能有真空包裝需要先拆除

### 4.2 可行方案

#### 方案 A：料匣式重力投放

```
┌─────────────────┐
│  飯塊料匣        │  ← 堆疊 20 塊，底部有擋板
│  ┌───┐          │
│  │飯塊│ ← 20   │
│  ├───┤          │
│  │飯塊│ ← 19   │
│  ├───┤          │
│  │...│          │
│  ├───┤          │
│  │飯塊│ ← 1    │
│  └───┘          │
│  ═══════════    │  ← 氣動擋板（氣缸推出/收回）
│     ↓           │
│   投入炒鍋       │
└─────────────────┘
```

- 原理：類似自動販賣機的落料機構
- 每次氣缸收回 → 底部飯塊落入鍋中 → 氣缸推出擋住下一塊
- **優點**：機構簡單、可靠、成本低
- **缺點**：飯塊間可能黏連；需要預拆包裝

#### 方案 B：傳送帶 + 推桿投放

```
飯塊 → [傳送帶] → [定位感測器] → [氣動推桿] → 落入鍋中
```

- 傳送帶定速運送，光電感測器偵測到飯塊位置後，氣缸推桿將其推入投料口
- 可搭配自動拆包機構

#### 方案 C：旋轉盤式（最推薦）

```
        ┌──────────┐
        │  旋轉盤   │
    ┌───┤ 6個飯塊槽 ├───┐
    │   └────┬─────┘   │
    │        │          │
    │    每轉一格        │
    │    底部開口對準     │
    │    投料口          │
    │        ↓          │
    │    飯塊落入鍋中     │
    └───────────────────┘
```

- 類似子彈轉輪的概念
- 6 個槽位，手動補充（或自動從料匣補充）
- 旋轉定位精確，一次投一塊
- Hestia Kitchen 採用類似的旋轉料倉（larder）概念

### 4.3 半解凍控制

| 方案 | 說明 |
|------|------|
| **料匣冷藏 + 定時取出** | 料匣維持 -18°C，出料前 30 分鐘移至 5°C 環境 |
| **恆溫料斗** | 料斗維持 -5°C，飯塊自然處於半解凍狀態 |
| **微波快速解凍** | 出料前短暫微波加熱（10-15 秒），不推薦（不均勻） |

**推薦**：恆溫料斗方案，設定 -5°C ~ 0°C，飯塊持續處於理想半解凍狀態。

## 七、技術挑戰與解決方案

### 7.1 冷凍方塊防黏連（最大挑戰）

| 解決方案 | 原理 | 可行性 | 成本 |
|----------|------|--------|------|
| **IQF 製程優化** | 供應商採用 IQF 急速冷凍，表面快速結凍不黏連 | ⭐⭐⭐⭐⭐ | 低（供應商端） |
| **防黏塗層噴霧** | 冷凍前噴灑食用油薄膜 | ⭐⭐⭐⭐ | 低 |
| **振動解塊器** | 持續低頻振動，防止靜止時黏連 | ⭐⭐⭐⭐ | 中 |
| **恆溫料斗（-5°C）** | 維持在稍高於冷凍但低於融化的溫度 | ⭐⭐⭐ | 中 |
| **間歇翻動** | 料斗內螺旋或翻動機構，定時攪動 | ⭐⭐⭐ | 中 |
| **乾冰/冷氣流** | 保持極低溫防止表面融化回凍 | ⭐⭐ | 高 |

**推薦組合**：IQF 製程（供應商端）+ 防黏塗層 + 振動料斗

### 7.2 投料口與炒鍋的物理對接

```
挑戰：
- 炒鍋溫度 280°C，產生大量蒸氣
- 蒸氣向上回流，會把投料口弄濕/結冰
- 油煙會沾黏投料管路

解決方案：

    投料口設計（截面圖）：
    
    ════════════  ← 防蒸氣擋板（不鏽鋼）
    ╔══════════╗
    ║ 投料滑道  ║  ← 45° 斜面，食材靠重力滑入
    ║          ║
    ╚════╗     ║
         ║     ║  ← 氣簾（壓縮空氣吹出，阻擋蒸氣上升）
    ─────╨─────╨──
    │    炒鍋    │
    └────────────┘

關鍵設計：
1. 氣簾（Air Curtain）：投料口下方吹出壓縮空氣，阻擋蒸氣回流
2. 斜面滑道：45° 角，食材靠重力快速通過
3. 擋板：物理隔離鍋內蒸氣
4. 快速投料：投料口只在需要時打開（氣動門）
```

### 7.4 投料管路的衛生清洗（CIP）

| 投料類型 | 清洗方式 | 頻率 |
|----------|---------|------|
| 固體滑道 | 高壓水沖洗 + 熱風乾燥 | 每日 |
| 蛋液管路 | **每次使用後**沖洗（蛋液易滋菌）| 每 2 小時 |
| 油管路 | 熱水 + 鹼性清潔劑沖洗 | 每日 |
| 醬油/水管路 | 清水沖洗 | 每日 |
| 料斗 | 拆卸清洗（設計為快拆式） | 每日 |

**蛋液管路特別注意**：
- 蛋液是高風險食材，5°C 以上細菌快速繁殖
- 管路全程需維持 5°C 以下
- 每 2 小時或每 30 份後自動沖洗
- 建議使用蠕動幫浦：**只需更換管路**即可完成「清洗」

### 7.5 食材在料斗中的保鮮

| 食材 | 料斗溫度 | 保存時限 | 補貨方式 |
|------|---------|---------|---------|
| 冷凍肉塊 | -5°C ~ 0°C | 8 小時 | 每日 3 次 |
| 冷凍蔬菜 | -5°C ~ 0°C | 8 小時 | 每日 3 次 |
| 冷凍飯塊 | -5°C ~ 0°C | 4 小時 | 每日 6 次（或更大料匣） |
| 蛋液 | 2°C ~ 5°C | 2 小時 | 小瓶頻繁更換 |
| 油/醬油 | 常溫 | 48 小時 | 每 2 天 |

## 八、對洪達的溝通建議

### 8.1 溝通重點

與洪達討論時，投料系統是**最需要深度對話的議題**。建議按以下順序討論：

#### 第一階段：了解洪達現有能力

1. **洪達現有的炒菜機，食材是怎麼放進去的？**
   - 是人工放入？還是有自動投料？
   - 調味料的投放是怎麼做的？（參考 BotInKit 的管路泵送方式）

2. **洪達有沒有做過固體食材的自動投料？**
   - 如果有，是什麼機構？什麼食材？
   - 如果沒有，他們對這個方向有什麼想法？

3. **洪達的炒鍋開口設計**
   - 開口多大？在頂部還是側面？
   - 有沒有預留投料口？
   - 蒸氣排放怎麼處理？

#### 第二階段：提出我們的需求

4. **展示 Cube Food 方塊食材的概念**
   - 帶實際的 2cm 方塊樣品去
   - 說明「標準化方塊讓自動化變簡單」的核心理念

5. **討論投料與炒鍋的整合方式**
   - 投料口的物理位置和尺寸
   - 是改造現有機型？還是需要重新設計？
   - 投料系統是洪達做？還是我們做？還是第三方做？

6. **時序控制的整合**
   - 炒鍋的控制系統能不能接受外部觸發訊號？
   - 還是投料時序由炒鍋內部程式控制？

#### 第三階段：討論分工

7. **投料系統的開發分工**

```
方案一：洪達全包
  洪達負責炒鍋 + 投料系統的整體設計
  Cube Rice 提供食材規格和配方需求
  → 最省心，但洪達可能沒有投料經驗

方案二：分工合作
  洪達負責炒鍋 + 控制系統
  第三方（食品機械廠）負責投料系統
  Cube Rice 負責系統整合
  → 最靈活，但整合挑戰大

方案三：Cube Rice 自建投料
  洪達提供炒鍋（含控制介面）
  Cube Rice 自行設計投料系統
  → 最有自主權，但需要機械設計能力
```
